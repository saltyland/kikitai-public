## 1. データベース（Supabase）

- [ ] 🔴 **マイグレーション反映**: `npm run sync` を実行して
      `20260705120000_replace_questions_tx.sql`（replace_questions RPC）を本番DBに適用する。
      未適用のままアプリだけデプロイすると **アンケートの保存が全て失敗する**。
- [ ] 🔴 **メール確認をON**: Authentication → Providers → Email → Confirm email を有効化。
      ※現状OFFのままだと他人のメールアドレスで無検証登録ができ、登録ボーナス100ptを無限に稼げる。
- [ ] 🔴 **リダイレクト許可URLの限定**: Authentication → URL Configuration で
      Site URL / Redirect URLs を本番ドメインのみに限定する。
- [ ] 🔴 **Leaked Password Protection を有効化**: Authentication → Policies（HaveIBeenPwned照合）。
- [ ] 🟡 バックアップ設定の確認（Point-in-Time Recovery が使えるプランか確認）。

## 2. ボット対策（Cloudflare Turnstile）

- [ ] 🔴 Cloudflare ダッシュボードで Turnstile サイトを作成し、サイトキー／シークレットキーを発行。
      https://dash.cloudflare.com/?to=/:account/turnstile
- [ ] 🔴 Vercel の環境変数に `NEXT_PUBLIC_TURNSTILE_SITE_KEY` / `TURNSTILE_SECRET_KEY` を設定。
      ※未設定だと検証はスキップされ、デモ開始（匿名アカウント作成）が無防備になる。

## 3. デプロイ（Vercel）

- [ ] 🔴 環境変数一式を Production に設定：
      `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY` /
      `GEMINI_API_KEY`（+ 任意で `GROQ_API_KEY` / `CEREBRAS_API_KEY` / `QUALITY_DEADLINE_MS`）/
      Turnstile 2種。※`LOCAL_LLM_*` は本番サーバにローカルLLMが無いので設定しない。
- [ ] 🔴 ローカルで `next build` が通ることを最終確認してから main へマージ→デプロイ。
- [ ] 🟡 **独自ドメインの取得・設定**（例: kikitai.jp / kikitai.app）。
      設定後、以下の3箇所のURLを新ドメインに更新すること：
      `app/layout.tsx`（metadataBase）/ `app/robots.ts`（sitemap URL）/ `app/sitemap.ts`（BASE）。
      未取得の間は kikitai.vercel.app のままで動く。
- [ ] ⚪ OGP画像（SNSシェア時のカード画像）の作成と `openGraph.images` への設定。
      現状は画像なしでも動作するが、SNS拡散を狙うなら効果大。

## リリース手順（上記が終わったら）

1. `release/public-launch` ブランチをコミット → PR 作成 → CI 緑を確認 → main へマージ。
2. `npm run sync`（マイグレーション反映）→ Vercel の Production デプロイ確認。
3. 本番URLでスモークテスト：
   - 新規登録（メール確認が届くか）→ オンボーディング → 回答 → ポイント付与
   - アンケート作成（通常公開・限定公開の両方）→ 公開 → 回答 → 結果表示 → CSV/xlsxダウンロード
   - `/help` `/privacy` `/operator` `/robots.txt` `/sitemap.xml` の表示確認
4. 問題なければ公開告知。
