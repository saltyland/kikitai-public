## 1. 技術スタック

- Next.js **16**（App Router / TypeScript / Turbopack）※従来のNextと破壊的変更あり（後述）
- Tailwind CSS v4

- Supabase CLI（`supabase` をdev依存に同梱、`npx supabase` で利用）

## 2. 環境・実行の前提（重要な落とし穴）

- **Node.js は PATH に通っていない**。実行時は毎回 PATH 追加が必要：
  - PowerShell: `$env:Path = "C:\Program Files\nodejs;" + $env:Path`
  - node: v24 / npm: v11
- **`.env.local` はUTF-8（BOMなし）**で保存すること。BOMが付くと先頭キーが壊れる。
- 開発サーバー：`npm run dev` → http://localhost:3000
- 型チェック：`npx tsc --noEmit` / ビルド：`npx next build`

### Next.js 16 の注意（AGENTS.md も参照）
- `cookies()` / `params` / `searchParams` は **非同期（Promise）**。`await` する。
- middleware は **`proxy.ts`** に改称（ルート直下）。本アプリの認証ガードは `proxy.ts` + `lib/supabase/proxy.ts`。
- コードを書く前に `node_modules/next/dist/docs/` の該当ガイドを読む方針。

## 3. アーキテクチャ（OOP）

UIから直接DBアクセスしない。層を分離：

```
app/                ページ＋ app/actions/（サーバーアクション）
components/          UI（クライアント/サーバー）
lib/
  supabase/         client(browser) / server / admin(service_role) / proxy
  services/         ビジネスロジック（AuthService 等）
  repositories/     DBアクセス（BaseRepository<T> を継承）
  domain/questions/ 設問タイプの多態（後述）
  types/
proxy.ts            認証ガード
supabase/migrations/  DBスキーマ（冪等マイグレーション）
scripts/supabase-sync.mjs  npm run sync の実体
```

- **設問タイプの多態**：`QuestionTypeDefinition`（抽象親）→ `ChoiceQuestionBase` → Single/Multiple/Scale、`TextQuestion`。`QuestionTypeRegistry` 経由でのみ利用。**新タイプ（順位付け・マトリクス等）はクラス追加＋レジストリ登録だけ**で済む設計（将来のポイント `pointCost` も保持）。
- **リポジトリ抽象基底**：`BaseRepository<T>`（共通CRUD）。

## 4. Supabase 運用

- スキーマは `supabase/migrations/` でバージョン管理（**冪等**＝再実行安全）。
- **手動でSQLをダッシュボードに貼らない**。`npm run sync` か `npm run db:push` で反映。
- `npm run sync` の中身（`scripts/supabase-sync.mjs`）：
  1. Management API で **メール確認OFF（mailer_autoconfirm=true）** を設定
  2. `supabase db push`（DBパスワードがあれば）
- 必要な秘密情報（`.env.local`・git管理外）：
  - `SUPABASE_ACCESS_TOKEN`（https://supabase.com/dashboard/account/tokens）
  - `SUPABASE_DB_PASSWORD`（db push に必要。未設定だとpushはスキップ）
  - `SUPABASE_SERVICE_ROLE_KEY`（退会でauth.usersごと削除する場合）
- プロジェクト参照（ref）：`cpyhkdlbwsiiwwucpkji`

### メール確認は OFF 済み
新規登録の `Invalid login credentials` 問題は **メール確認をOFFにして解決済み**
（ダッシュボード＋ `npm run sync` の両方で適用）。登録すると即ログイン状態でホームに入る。
※ OFF前に作った「未確認アカウント」は Authentication → Users から削除して登録し直す。
